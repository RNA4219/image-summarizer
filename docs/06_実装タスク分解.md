# 実装タスク分解 v0.1

## 1. 目的

本書は、[01_要件定義.md](./01_要件定義.md)、
[02_仕様書.md](./02_仕様書.md)、
[04_APIインターフェース定義.md](./04_APIインターフェース定義.md)、
[05_画面仕様書.md](./05_画面仕様書.md)
をもとに、実装作業を着手可能な単位まで分解するための文書である。

本書では、以下を明確にする。

- Phase ごとの実装範囲
- フロントエンド / バックエンド / 連携 / テストの担当範囲
- 各タスクの完了条件
- 依存関係
- 実装順の推奨

## 2. 実装方針

実装は以下の原則で進める。

- まず `api` OCR + `api` 要約の既存経路を壊さずに、モード切替の枠組みを入れる
- `Ollama` および `NDLOCR-Lite` は抽象化ポイントを先に作る
- 実接続は Phase 2 で行う
- UI は先にラジオボタンと結果表示の受け皿を整える
- テストは API 契約変更に追従させる

## 3. Phase 構成

| Phase | 目的 | 主な成果物 |
| --- | --- | --- |
| Phase 1 | モード切替の土台整備 | API 契約反映、UI 選択追加、抽象化導入 |
| Phase 2 | ローカル連携実装 | Ollama 連携、NDLOCR-Lite 連携 |
| Phase 3 | UX / 精度改善 | 推奨モード表示、後処理改善、運用整備 |

## 8. Phase 1 完了条件

Phase 1 完了とみなす条件は以下とする。

- OCR 方式と要約方式を UI から選べる
- API が `ocr_mode` / `summary_mode` を受け取れる
- `api` / `api` の既存経路が維持されている
- `local_llm` / `high_accuracy` は未接続でも明示エラーを返せる
- 結果表示に `ocrMode` / `summaryMode` が含まれる
- API テストが新契約に追従している
- フロントエンドがビルド可能である
