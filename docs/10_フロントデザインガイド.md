# フロントデザインガイド v0.1

## 1. 目的

本書は、画像要約アプリのフロントエンド実装において、
青と白を基調にしたモダンな UI を一貫して実装するためのデザインガイドである。

本書では以下を定義する。

- デザインコンセプト
- カラーパレット
- タイポグラフィ方針
- レイアウトと余白
- カード / ボタン / フォームのトーン

## 2. デザインコンセプト

### キーワード

- Clean
- Trustworthy
- Modern
- Calm
- Readable

### 目指す印象

- 業務利用でも違和感のない清潔感
- AI アプリとしての先進性
- 情報量が多くても読み疲れしない整理感

### 避けるもの

- 彩度の高すぎる配色
- 色数が多すぎる状態表現
- 過度に装飾的なグラデーション
- 暗すぎるダーク寄り UI

## 3. カラーパレット

## 3.1 基本色

| 役割 | 色コード例 | 用途 |
| --- | --- | --- |
| Primary 700 | `#0F4C81` | 見出し、強いアクセント |
| Primary 600 | `#1565C0` | 主ボタン、リンク、アクティブ状態 |
| Primary 500 | `#1E88E5` | 補助アクセント |
| Primary 100 | `#DCEEFF` | 選択状態の背景、淡色チップ |
| Surface | `#FFFFFF` | カード、フォーム背景 |
| Surface Alt | `#F7FAFD` | ページ背景、区切り背景 |
| Border | `#D7E3F1` | 枠線、区切り線 |
| Text Primary | `#16324A` | 本文、見出し |
| Text Secondary | `#5E7488` | 補助説明文 |

## 3.2 状態色

| 状態 | 色コード例 | 方針 |
| --- | --- | --- |
| Success | `#2E7D32` | 成功通知、成功チップ |
| Warning | `#ED8B00` | 注意表示、不確実情報 |
| Error | `#C62828` | エラー表示 |

### 状態色の原則

- 状態色は必要箇所に限定して使う
- UI の主印象は青と白のまま維持する
- エラー以外の多用は避ける

## 4. 背景と面構成

### 4.1 ページ背景

- 純白またはごく薄い青みを帯びた白を使う
- 単色でフラットにしすぎず、わずかな色差で面を分ける

### 4.2 カード

- 白背景
- 角丸は中程度
- 影は薄く短く
- 境界線をうっすら入れて整える

### 4.3 セクション分離

- 強い罫線より余白で分ける
- 必要な箇所だけ薄いボーダーを使う

## 5. タイポグラフィ

### 5.1 方針

- 可読性を最優先する
- タイトルは力強く、本文は静かに
- 日本語が詰まって見えないように行間を確保する

### 5.2 階層

| レベル | 用途 | 方針 |
| --- | --- | --- |
| H1 | 画面タイトル | 太め、青系アクセント可 |
| H2 | セクション見出し | 明確な区切り |
| Body | 説明文 / 要約本文 | 最優先で読みやすく |
| Caption | 補足、メタ情報 | 控えめな濃度 |

### 5.3 文字色

- 見出しは `Text Primary`
- 補助説明は `Text Secondary`
- エラー文は `Error`

## 6. 余白と密度

### 6.1 原則

- 余白で整理する
- 1 画面に情報を詰め込みすぎない
- フォーム部と結果部の間隔を十分に取る

### 6.2 推奨リズム

- セクション間: 広め
- フォーム項目間: 中程度
- ラベルと入力要素: 近め
- 結果カード間: 中程度

## 7. ボタン

### 7.1 主ボタン

- 青系塗り
- 白文字
- 横幅は必要十分
- ローディング時もレイアウトが崩れない

### 7.2 副ボタン

- 白背景
- 青系枠または青系文字
- 主ボタンの邪魔をしない

### 7.3 非活性

- 彩度を落とし、操作不可が明確に分かる
- ただし、情報が消えたように見せない

## 8. フォーム

### 8.1 ラジオボタン

- ラベルテキストを主役にする
- 選択状態は青で明示する
- モバイルでは縦並び優先でもよい

### 8.2 ファイル選択部

- 操作入口が目立つようにする
- 選択済みファイルはチップまたはカードで整理する
- 非対応形式はエラー色で区別する

## 9. 結果表示

### 9.1 単一結果

- 要約文を最も目立たせる
- 利用モードはチップで静かに補足する
- 詳細情報と抽出テキストは折りたたみで情報量を制御する

### 9.2 複数結果

- サマリー行で全体の成否を把握できるようにする
- 個別カードは同一構造で並べる

## 10. 動きとフィードバック

- ローディングは控えめにする
- 派手なアニメーションは不要
- ボタン、展開、結果表示に必要最小限のフィードバックを入れる

## 11. 実装時の優先順位

1. 配色の一貫性
2. 余白とカード整理
3. 文字の読みやすさ
4. 状態表示の明確さ
5. 装飾
